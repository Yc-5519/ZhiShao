# 智哨 - 基于 RDK X5 的隐私友好型智能看护系统

智哨是一套围绕 RDK X5 边缘计算开发板构建的智能看护系统。系统结合本地人体姿态检测、舵机云台跟随、云端视觉大模型、网页看护页面和飞书提醒，为居家看护场景提供更及时、更克制、更尊重隐私的安全辅助能力。

项目面向老人独自在家、家属无法长期陪伴的场景。智哨并不把“持续展示真实画面”作为默认方式，而是优先使用骨骼图、状态摘要和隐私复核结果，让家属了解安全状态，同时减少对老人生活细节的暴露。

## 核心能力

- 在 RDK X5 上运行 YOLOv8 Pose 人体姿态检测。
- 默认展示脱敏骨骼图，而不是持续展示真实摄像头画面。
- 基于人体关键点进行摔倒风险判断、事件记录和飞书提醒。
- 使用舵机控制云台，根据目标位置调整摄像头方向，减少固定视角盲区。
- 支持人物锁定、自动跟随和短时丢失后的目标接回。
- 接入 DashScope 平台 Qwen-VL 视觉大模型，用于隐私复核、画面理解、问答和日报总结。
- 通过云服务器提供公网看护入口和用户登录能力。
- 通过飞书管家提供系统自检、监控链接、日报、最近事件和看护指令。
- 用户账号由云服务器管理，密码以哈希形式保存，不存放在 RDK 设备上。

## 系统架构

```text
居家环境
  -> 摄像头
  -> RDK X5 边缘端
       - YOLOv8 Pose 人体姿态检测
       - 脱敏骨骼图生成
       - 摔倒风险状态机
       - 舵机云台控制
       - 看护事件记录
  -> 云端网关
       - 公网入口
       - 用户登录
       - 反向隧道访问 RDK 看护页面
  -> 云端视觉大脑
       - DashScope / Qwen-VL
       - 隐私复核
       - 画面理解
       - 问答与总结
  -> 用户端
       - Web 看护页面
       - 飞书管家
       - 手机浏览器访问
```

## 运行组成

### RDK X5 边缘端

RDK 主程序负责摄像头采集、YOLOv8 Pose 推理、目标跟随、摔倒风险判断、舵机云台控制、事件记录、网页看护页面和飞书指令处理。

RDK 正式运行目录：

```text
/home/sunrise/ZhiShao_V2
```

RDK 正式服务：

```text
zhishao-rdk
```

### 云端登录网关

云端登录网关负责公网访问和用户身份验证。用户在访问公网看护页面前需要登录，账号由管理员在云服务器上创建和管理。

主要组件：

```text
cloud_auth/
nginx
zhishao-auth
```

### 云端视觉大脑

云端视觉大脑通过 DashScope Qwen-VL 提供复杂画面理解能力。RDK 通过本地转发地址访问云端服务，云端模型服务端口不直接暴露到公网。

主要链路：

```text
RDK brain_client.py
  -> http://127.0.0.1:19000
  -> SSH 隧道
  -> 云服务器 127.0.0.1:9000
  -> DashScope / Qwen-VL
```

## 用户交互

### Web 看护页面

网页端提供：

- 脱敏骨骼看护画面
- 当前安全状态
- 人物锁定与跟随状态
- 舵机云台控制
- 最近看护事件
- 日报入口
- 临时查看真实画面的隐私确认流程

### 飞书管家

常用指令：

```text
系统自检
监控链接
说明书
现在是否安全
最近事件
日报
锁定当前人物
取消锁定
暂停跟随
恢复跟随
云台回中
临时查看真实画面
关闭真实画面
```

当用户询问“现在是否安全”时，系统会先说明当前安全状态，再说明今天是否出现过高风险提醒，并在有可用证据时附带脱敏骨架 GIF。

## 隐私设计

智哨的默认设计是“少打扰、可确认”：

- 默认不展示真实摄像头画面。
- 默认使用骨骼图、目标状态和事件摘要。
- 临时查看真实画面需要用户明确确认。
- 配置云端隐私复核后，真实画面开启前会先经过复核。
- 飞书回复优先发送文字结论和脱敏骨架证据。
- 人物锁定不依赖人脸识别。

## 项目目标

智哨希望把看护从“被动查看”变成“主动发现”。系统通过本地识别及时发现风险，通过舵机云台减少视野盲区，通过网页和飞书让家属及时了解状态，并通过脱敏显示和隐私复核减少不必要的真实画面暴露。
