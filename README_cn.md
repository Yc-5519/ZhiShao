# 智哨 - 基于 RDK X5 的隐私友好型智能看护系统

智哨是一套面向居家老人看护场景的智能看护系统。系统以 RDK X5 作为边缘计算核心，结合本地人体姿态检测、舵机云台跟随、云端视觉理解、网页看护页和飞书提醒，帮助家属更及时地了解老人安全状态。

项目同时关注两个核心问题：

- 老人摔倒、异常姿态、摄像头异常、长时间无人等情况需要及时发现。
- 老人不希望被持续监控，真实摄像头画面不应默认暴露。

## 核心能力

- 在 RDK X5 上本地运行 YOLOv8 Pose 人体姿态检测。
- 默认显示脱敏骨架图，而不是持续显示真实摄像头画面。
- 根据人体关键点判断疑似摔倒和异常姿态，并记录看护事件。
- 对确认的高风险情况和设备异常通过飞书提醒家属。
- 使用舵机云台根据目标位置调整摄像头方向，减少固定视角盲区。
- 支持人物锁定、短时目标接回和重复目标抑制。
- 提供适配手机访问的网页看护页，并通过云端网关进行登录保护。
- 接入 DashScope 平台的 Qwen-VL Max，用于隐私复核、画面理解、问答和日报总结。
- 用户账号由云服务器管理，密码以哈希形式保存，不存放在 RDK 设备上。

## 系统组成

```text
摄像头与舵机云台
  -> RDK X5 边缘端主程序
       -> YOLOv8 Pose 人体姿态检测
       -> 脱敏骨架图生成
       -> 摔倒与异常状态判断
       -> 人物跟随与云台控制
       -> 看护事件记录
       -> 本地网页看护页与飞书指令

RDK X5
  -> SSH 隧道
  -> 云端网关与云端视觉服务
       -> 用户登录
       -> 公网访问
       -> DashScope / Qwen-VL Max

家属端
  -> 手机或电脑浏览器
  -> 飞书管家
```

## 运行组件

### RDK X5 边缘端

RDK 主程序负责摄像头采集、YOLOv8 Pose 推理、骨架图生成、摔倒风险判断、人物跟随、舵机云台控制、事件记录、网页看护页和飞书指令处理。

RDK 正式运行目录：

```text
/home/sunrise/ZhiShao_V2
```

RDK 正式 systemd 服务：

```text
zhishao-rdk
```

### 云端登录网关

云端登录网关负责公网访问和用户身份验证。用户访问公网看护页前需要登录，账号由管理员在云服务器上创建和管理。

主要组件：

```text
cloud_auth/
nginx
zhishao-auth
```

### 云端视觉服务

云端视觉服务通过 DashScope Qwen-VL Max 提供更复杂的画面理解能力。RDK 通过本地转发地址访问云端服务，云端模型服务端口不直接暴露到公网。

主要链路：

```text
RDK brain_client.py
  -> http://127.0.0.1:19000
  -> SSH 隧道
  -> 云服务器 127.0.0.1:9000
  -> DashScope / Qwen-VL Max
```

## 用户交互

### Web 看护页

网页端提供：

- 脱敏骨架看护画面
- 当前安全状态
- 目标 ID 与锁定状态
- 舵机云台控制按钮
- 摔倒和异常事件记录
- 最近事件
- 日报入口
- 临时查看真实画面的隐私复核流程

### 飞书管家

常用指令：

```text
说明书
系统自检
监控链接
当前是否安全
当前动作
最近事件
日报
锁定当前人物
取消锁定
暂停跟随
恢复跟随
云台回中
临时查看真实画面
关闭真实画面
```

当用户询问“当前是否安全”时，飞书管家会先说明当前状态是安全还是危险，再说明今天是否出现过高风险提醒。有可用证据时，会附带脱敏骨架 GIF，而不是直接暴露真实摄像头画面。

## 隐私设计

智哨的默认设计是“少打扰、可确认”：

- 默认不显示真实摄像头画面。
- 默认使用骨架图、目标状态和事件摘要。
- 临时查看真实画面需要用户明确确认。
- 开启真实画面前可先经过云端隐私复核。
- 飞书回复优先发送文字结论和脱敏骨架证据。
- 人物锁定不依赖人脸识别。

## 安全与设备异常处理

系统会记录并提醒以下看护情况：

- 疑似摔倒或高风险姿态
- 摄像头断开或无法读取画面
- 长时间没有看到可信人体目标
- 重新看到目标后的恢复事件
- 公网入口或服务连通性异常

对于摔倒相关事件，系统优先提醒家属确认安全，并提供简洁状态和可用的脱敏证据。

## 项目目标

智哨希望把看护从“被动查看”变成“主动发现”。系统帮助家属及时了解老人是否安全，同时通过脱敏显示和隐私复核，减少不必要的真实画面暴露，尽量保留老人的生活尊严。
